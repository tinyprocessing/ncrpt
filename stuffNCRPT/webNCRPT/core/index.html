How It Works

Our system allows you to flexibly set up file protection in your company. You can set up your own server using our system.

Protecting a file Decrypting a file
The file is imported into the application, our system performs local analysis.
The employee specifies a list of users and rights for protection. The data comes from your server, access synchronization
The system creates an AES key, the key is used to encrypt the license and the file itself. without leaving your application.
Application stores ciphertext (instead of clear text) in storage.
NCRPT signs the license and with your organization's public key, the protected file is formatted.
The client is authorized in the application
The application sends a request for a license on user authorization
The program decrypts the file with the AES key
The file is added to the protected space for display without leaving your application.
The user can work with the file based on the rights set for him

Reduce breach risk, by better protecting sensitive data

Encrypt data before it leaves your application, so the storage layer, admins, and adversaries only ever see ciphertext.
